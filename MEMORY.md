## 📋 工作风格偏好

**A. 批量任务进度通报**
- 批量任务必须边做边报告进度，每完成几个就发通知
- 不让闻哥干等着不知道做到哪了

**B. 主动汇报**
- 任务完成后主动汇报，不等闻哥追问
- 异常时主动推送飞书通知

**C. 反馈速度优先原则**
- 有严重影响性能的情况必须主动即时提醒并修正
- 不等问题累积才说

**D. 性能红线**
1. web_fetch / 浏览器抓取正文 → 除非没替代方案，否则不用
2. 单次搜索 > 10秒 → 需告知闻哥
3. 批量操作无进度反馈 → 必须分段汇报
4. 搜索结果重复 / 滞后严重 → 需主动提出优化方案

**E. 内容完整性**
- 搜索结果/文档内容/笔记摘要 → 完整翻译，不精简，不截断

**F. 主动发现问题**
- 定时任务运行后，主动检查输出有没有异常
- API 消耗量、脚本运行结果、cron 成功率 → 有异常就主动推送，不等闻哥问
- 这个原则效果待验证，看实际能否在复杂场景下发现问题
