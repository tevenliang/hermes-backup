# Douyin: Why yt-dlp Cannot Work

## The Error

Every yt-dlp invocation against a Douyin URL produces this result:

```
[generic] Extracting URL: https://v.douyin.com/ZdbZCAHFvfI/
[generic] ZdbZCAHFvfI: Downloading webpage
Extracting cookies from chrome
Extracted 3397 cookies from chrome
[redirect] Following redirect to https://www.douyin.com/video/7634760396226336116
[Douyin] Extracting URL: https://www.douyin.com/video/7634760396226336116
[Douyin] 7634760396226336116: Downloading web detail JSON
WARNING: [Douyin] 7634760396226336116: Failed to parse JSON: Expecting value in '': line 1 column 1 (char 0)
ERROR: [Douyin] 7634760396226336116: Fresh cookies (not necessarily logged in) are needed
```

The error message is misleading. It implies the cookies are stale or missing — but they are not.

## Root Cause

yt-dlp's Douyin extractor (in `yt_dlp/extractor/tiktok.py`, `DouyinIE._real_extract`) calls:

```python
'https://www.douyin.com/aweme/v1/web/aweme/detail/'
```

This endpoint requires a `s_v_web_id` cookie value that is **dynamically generated by Douyin's JavaScript running in a real browser**. yt-dlp (a Python HTTP client) cannot execute JavaScript, so it cannot obtain this value.

The extractor itself acknowledges this in a TODO comment:
```python
# TODO: Run verification challenge code to generate signature cookies
```

The extractor checks for `s_v_web_id` and raises the "Fresh cookies" error when it's absent or invalid — regardless of whether the user is logged in.

## What Doesn't Work

| Method | Result |
|--------|--------|
| `--cookies /path/to/cookies.txt` (Netscape format) | Same error |
| `--cookies-from-browser chrome` (reads 3397 cookies from Chrome SQLite) | Same error |
| EditThisCookie JSON export → converted to Netscape | Same error |
| Chrome DevTools → Application → Cookies → Export JSON | Same error |
| Adding `--add-header` for User-Agent, Accept-Language | Same error |
| Updating yt-dlp to latest version (`yt-dlp -U`) | Same error |
| `--extractor-args douyin:client_version=...` | Same error |

## Verified Approaches

### Option A: yt-dlp (Does NOT work)
No combination of cookies, headers, or extractor args overcomes this. The `s_v_web_id` is generated per-session by JavaScript and expires quickly — it cannot be exported or replicated.

### Option B: agent-browser (Works — Use this)
Control a real Chrome browser that is already logged into Douyin. Chrome executes the JavaScript naturally, so all API calls work.

```bash
# Using agent-browser skill
agent-browser navigate "https://www.douyin.com/video/7634760396226336116"
agent-browser screenshot
# Then use OCR/vision to extract text from screenshot
# Or use browser's JavaScript access to extract video/subtitle data
```

**Steps**:
1. Open Chrome on the user's machine, navigate to the Douyin video
2. Use `agent-browser` to take a screenshot or extract DOM content
3. Feed the screenshot to `mcp_minimax_understand_image` for OCR
4. Or use browser's JavaScript console to call Douyin's internal API directly (returns JSON with video URLs)

### Option C: OpenWhisper (Partial — video download still blocked)
Even if you could download the video file, Douyin's video stream requires the same JavaScript signature for the stream URL. Whisper can transcribe only if you have a local video file — and getting that file requires Option B or screen recording.

## Technical Reference

- yt-dlp source: `/opt/homebrew/Cellar/yt-dlp/2026.3.17/libexec/lib/python3.14/site-packages/yt_dlp/extractor/tiktok.py`
- Relevant line (1486): `expected=not self._get_cookies(self._WEBPAGE_HOST).get('s_v_web_id')`
- The error is raised when `detail` is None after `_download_json` returns empty
- `s_v_web_id` is set by Douyin's `generateSignature` JavaScript function

## Recommendation

For any task involving Douyin video transcription or content extraction: use `agent-browser` to control Chrome directly. This is the only reliable path.
